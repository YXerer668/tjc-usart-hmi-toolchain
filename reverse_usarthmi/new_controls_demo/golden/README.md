This folder records the first live-verified mixed-control TFT generated by the local
USART HMI toolchain.

- Port: COM36
- Screen: TJC8048X543_011C
- App baud: 9600
- Upload baud: 921600
- Upload chunk size: 4096 bytes
- Verified at: 2026-05-12T22:12:53+08:00

Runtime smoke command without re-uploading:

```powershell
python tools\live_tft_smoke.py `
  --file reverse_usarthmi\new_controls_demo\local_build\output.tft `
  --out-dir reverse_usarthmi\new_controls_demo\golden\last_smoke `
  --expect-json reverse_usarthmi\new_controls_demo\golden\expected_runtime.json `
  --port COM36 --baud 9600 --timeout-ms 3000
```

Runtime smoke command with upload:

```powershell
python tools\live_tft_smoke.py `
  --upload --progress `
  --file reverse_usarthmi\new_controls_demo\local_build\output.tft `
  --out-dir reverse_usarthmi\new_controls_demo\golden\last_smoke_upload `
  --expect-json reverse_usarthmi\new_controls_demo\golden\expected_runtime.json `
  --port COM36 --baud 9600 `
  --download-baud 921600 --chunk-size 4096 `
  --timeout-ms 3000 --prepare-delay-ms 1000 --prepare-wait-ms 800
```

Known rule from live testing:

- Public `whmi-wri` upload is ACKed at 4096-byte chunks on this screen.
- A 2048-byte chunk probe entered update mode and waited for the rest of the 4096-byte block.
- Keep normal uploads at `--chunk-size 4096`.
